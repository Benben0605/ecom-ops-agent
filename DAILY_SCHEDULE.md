# 每日时间分配建议（8-10h 强度）

> 配套 PROGRESS.md 使用。核心不是"塞满时长"，而是把精力花在对的地方、并留出吸收时间。

## 三条黄金原则
1. **最难的放上午**：精力最好的时段做当天最硬的核心任务（如手写 Agent loop），别拿来回邮件/调样式。
2. **必须留"消化 + 输出"时间**：你是教程级，吸收速度才是真瓶颈。手写过去 ≠ 学会，每天结尾要能把代码讲出来。
3. **每天有明确收工 review**：高强度下没有复盘 = 瞎忙。每天贴进度对齐一次。

## 标准日（约 8h 硬核 + 1h 弹性）

| 时段 | 时长 | 内容 | 说明 |
|---|---|---|---|
| 09:00–09:20 | 20m | **启动** | 看 PROGRESS，定今天**唯一主目标**，拆成 2-3 个里程碑 |
| 09:20–12:00 | 2h40 | **主线开发·深水区** | 当天最难的核心任务，深度专注，关消息 |
| 12:00–13:30 | — | 午休 | 真休息，给大脑后台消化上午的内容 |
| 13:30–15:30 | 2h | **主线开发·续 + 联调** | 把上午的成果跑通、串起来 |
| 15:30–15:45 | 15m | 休息 | |
| 15:45–16:45 | 1h | **攻坚缓冲 / 查漏** | 补没搞定的、清理代码、处理卡点 |
| 16:45–17:45 | 1h | **源码精读（learn-repo）** | 读 Claude Code / 框架对应模块，反哺今天的设计 |
| 17:45–18:45 | 1h | **每日 review + 沉淀** | 更新 PROGRESS、贴进度给我对齐、把当天代码讲一遍、记下没懂的点 |
| 弹性 +1h | 1h | **八股 / 写笔记（可选）** | 用 tutor 自测或写技术笔记；状态好才做，不硬撑 |

## 一周内的节奏微调
- **Week 1**（学新东西最多）：源码精读 + 消化时间加重，开发别贪快，宁可少写多懂。
- **Week 2**（工程化部署）：开发块拉满，review 照常。
- **Week 3**（评估 + 迭代）：下午多留"分析/思考"而非纯码——优化是想出来的不是堆出来的。
- **Week 4**（求职转化）：开发降到一半，另一半给简历 / 博客 / 模拟面试 / 投递。

## 两条红线
- **单点卡死超 45 分钟** → 停，写进 PROGRESS 卡点，问我；别自己硬磕一下午。
- **23:00 前收工**，每周留半天彻底休息或机动——睡眠和休息是吸收的一部分，全月别 burnout。
